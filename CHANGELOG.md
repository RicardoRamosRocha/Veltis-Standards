# Changelog

Todas as mudanças relevantes do Veltis Standards devem ser registradas neste arquivo.

## [0.3.0] - 2026-06-26

### Adicionado

- Versão 1.0 inicial completa do módulo Git em `standards/git`.
- Guias oficiais para Git, Git Flow, GitHub Flow, branches, commits, pull requests, tags, releases, conflitos de merge e sincronização entre máquinas.
- Templates básicos e avançados para mensagens de commit, pull requests, releases, nomenclatura de branches, workflows Git, configuração GitHub e múltiplos ambientes.
- Exemplos básicos, exemplos aplicados a projetos reais, estudos de caso, ADRs e release notes do módulo Git.

### Alterado

- Roadmap e backlog globais atualizados para refletir a conclusão do módulo Git v1.
- Auditoria global atualizada para registrar Git como módulo crítico concluído em versão 1.0 inicial.

## [0.2.0] - 2026-06-26

### Adicionado

- Versão 1.0 inicial completa do módulo Documentation em `standards/documentation`.
- Guias, templates, exemplos, estudo de caso, ADR e release notes do padrão oficial de documentação.
- Relatório de auditoria `AUDIT.md` para preparação da futura Release 1.0.

### Alterado

- Padronização de READMEs, MANIFESTs, roadmaps e backlogs dos módulos.

## [0.1.0] - 2026-06-26

### Adicionado

- Estrutura inicial do repositório Veltis Standards.
- Documentação central em `docs/` para visão de produto, requisitos, backlog, roadmap, arquitetura e organização.
- Módulos iniciais em `standards/` para documentação, Git, arquitetura, ASP.NET Core, banco de dados, UI, segurança, testes, DevOps, Flutter, IA e checklists.
- Arquivos principais por módulo: `README.md`, `MANIFEST.md`, `CHANGELOG.md`, `roadmap.md` e `backlog.md`.
- Subpastas padronizadas por módulo para documentação, templates, exemplos, estudos de caso, ADRs, releases e assets.

### Definido

- Princípios de organização, governança e evolução do framework.
- Critérios iniciais para contribuições, revisões e publicação de alterações.
