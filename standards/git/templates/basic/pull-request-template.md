# Template de Pull Request

## Resumo

Descreva o que foi alterado e por quê.

## Tipo de mudança

- [ ] Funcionalidade
- [ ] Correção
- [ ] Documentação
- [ ] Refatoração
- [ ] Testes
- [ ] Segurança
- [ ] Manutenção

## Escopo

Liste módulos, telas, APIs, documentos ou fluxos afetados.

## Validação

Descreva comandos executados, testes realizados ou revisão manual aplicada.

```bash
# exemplo
git diff --check
```

## Riscos

Liste riscos conhecidos, impactos operacionais ou pontos de atenção.

## Checklist

- [ ] Branch atualizada com a base correta.
- [ ] Commits com mensagens claras.
- [ ] Documentação atualizada quando necessário.
- [ ] Testes ou validações aplicáveis executados.
- [ ] Nenhum segredo, token, senha ou dado sensível foi incluído.
