# Guia de Estilo Markdown

## Objetivo

Este guia define convenções de escrita em Markdown para documentos da Veltis. O foco é tornar os arquivos legíveis em editores, plataformas Git e ferramentas de revisão.

## Idioma e tom

- Escreva em português.
- Use linguagem profissional, direta e objetiva.
- Evite termos vagos quando puder indicar comportamento esperado.
- Prefira voz ativa.
- Explique siglas na primeira ocorrência quando não forem universais.

## Títulos

- Use um único título de nível 1 por arquivo.
- Organize seções principais com `##`.
- Use `###` para subdivisões quando necessário.
- Evite pular níveis de título.

## Listas

Use listas para critérios, etapas e responsabilidades. Evite listas longas sem agrupamento lógico.

## Tabelas

Use tabelas para comparar informações estruturadas, como status, responsáveis, versões ou decisões. Evite tabelas para texto longo.

## Links

Prefira links relativos dentro do repositório. Verifique links antes de publicar uma alteração.

## Blocos de código

Use blocos de código apenas para comandos, exemplos de configuração, contratos ou trechos que precisem preservar formatação. Não inclua segredos ou valores reais.

## Ênfase

Use negrito com moderação para termos importantes. Evite excesso de destaque visual.

## Datas e versões

Use datas no formato `AAAA-MM-DD`. Use versionamento consistente com o projeto ou módulo.

## Checklists

Use checklists quando a intenção for revisão ou acompanhamento operacional.

```markdown
- [ ] Critério pendente
- [x] Critério atendido
```

## Segurança

Antes de publicar, revise se o arquivo contém dados reais, credenciais, URLs sensíveis, nomes de ambientes restritos ou informações que não deveriam estar no repositório.
