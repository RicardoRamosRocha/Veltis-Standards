# ADR-001 - Adotar documentação em Markdown

## Status

Aceito

## Data

2026-06-26

## Contexto

A equipe precisa manter documentação versionada junto ao repositório do projeto, permitindo revisão em pull requests e histórico de mudanças.

## Decisão

Adotar Markdown como formato padrão para documentação inicial do projeto.

## Alternativas consideradas

- Documento externo em editor colaborativo: facilita edição visual, mas reduz rastreabilidade junto ao código.
- Wiki separada: organiza navegação, mas pode se desatualizar em relação ao repositório.

## Consequências

### Positivas

- Documentação versionada junto ao projeto.
- Revisão documental no mesmo fluxo de pull request.
- Baixa barreira de edição.

### Negativas

- Recursos visuais avançados exigem disciplina adicional.
- Navegação depende da organização dos arquivos.

## Revisão

Revisar se o volume documental exigir uma plataforma complementar de publicação.
