# Roadmap - Architecture

## Objetivo

Evoluir o módulo Architecture como o padrão oficial de arquitetura de software da Veltis, mantendo orientações práticas para desenho, documentação, revisão e evolução de sistemas.

## Versões

### Versão 1.0

#### Entregas

- Publicar guias de arquitetura, camadas, Clean Architecture, DDD e padrões estruturais.
- Publicar guias de DI, Repository Pattern, Service Layer, CQRS e monólito modular.
- Criar templates básicos e avançados.
- Criar exemplos básicos e exemplos aplicados a projetos reais.
- Registrar estudos de caso, ADRs e release notes.

#### Critérios de conclusão

- Arquivos principais revisados e coerentes com a versão 1.0.
- Guias, templates, exemplos, estudos de caso, ADRs e release notes disponíveis.
- Conteúdo escrito em português, conceitual, aplicável e sem código sensível.

#### Próximos passos

- Coletar feedback de uso em revisões arquiteturais reais.

### Versão 1.1

#### Entregas

- Criar checklists de revisão arquitetural por tipo de projeto.
- Adicionar exemplos de diagramas arquiteturais.
- Criar matriz de escolha entre camadas, Clean Architecture e monólito modular.

#### Critérios de conclusão

- Checklists aplicáveis em revisões técnicas.
- Exemplos revisados para projetos pequenos, SaaS e corporativos.
- Changelog atualizado com mudanças relevantes.

#### Próximos passos

- Avaliar necessidades de governança e auditoria arquitetural.

### Versão 1.2

#### Entregas

- Definir critérios de maturidade arquitetural.
- Criar estudos de caso adicionais.
- Refinar critérios de exceção e revisão de ADRs.

#### Critérios de conclusão

- Critérios de maturidade claros e revisáveis.
- Estudos de caso suficientes para orientar adoção.
- Exceções documentadas e rastreáveis.

#### Próximos passos

- Preparar indicadores e automações para a versão 2.0.

### Versão 2.0

#### Entregas

- Integrar critérios arquiteturais a fluxos de revisão e entrega quando aplicável.
- Definir indicadores de aderência arquitetural.
- Consolidar governança de revisão periódica de arquitetura.

#### Critérios de conclusão

- Indicadores definidos.
- Processo de revisão arquitetural documentado.
- Automação mínima especificada ou implementada quando fizer sentido.

#### Próximos passos

- Planejar evolução pós-2.0 com base em métricas, auditorias e feedback das equipes.
